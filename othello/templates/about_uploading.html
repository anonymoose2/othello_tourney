File specification

A valid Othello AI consists of file called strategy.py with a class called Strategy containing a method called best_strategy taking the arguments board, player, best_move, still_running in that order. best_move and still_running are multiprocessing.Value objects, and can be changed using best_move.value=34 for example. still_running is another multiprocessing.Value flag, initially set to 1, then set to 0 by the server when the time limit for the AI is almost up.

The board will be a length-100 string of characters from the set {'?', '.', '@', 'o'}. This represents an 8x8 board of blanks '.' surrounded by a border '?'. The player strings are '@' for black and 'o' for white. A move is an integer representing the index in the board string where the current player will place a piece.

Example code

To get you started, here is the most basic form of AI, random choice: 


import random
import time

class Strategy():
    # implement all the required methods on your own
    def best_strategy(self, board, player, best_move, running):
        time.sleep(1)
        if running.value:
            best_move.value = random.choice(self.legal_moves(board, player))

Multiple files

As of now, you can only upload 1 file, strategy.py. You should try putting all your code in that file if possible. Any files created by your code will persist between runs and games. If you do in fact need other files for you code to work, and it is infeasible to include it in one file, contact othello@tjhsst.edu or your AI teacher about your situation.

Other Info

If you code fails to return a move, returns an invalid move, or errors for whatever reason, it will be treated as a forfeit. As such, make sure you code works on your computer before using this website to test against other AIs. Just as a reminder, if your code is caught cheating during the tournament in any way, you will be caught and given an integrity violation, so just don't do it.
